### EPIC A — Android App Shell (Kotlin) + Foreground Service
**Goal:** A service-only Android app that can start/stop the native proxy and stay alive with a foreground notification.

#### Story A1 — Create Android project structure
- Task A1.1: Create new Android app project (minSdk/targetSdk decided; start with API 30+).
- Task A1.2: Add required permissions in manifest (`INTERNET`, `FOREGROUND_SERVICE`).
- Task A1.3: Add service declaration in manifest (service exported = false).
- Task A1.4: Add (optional) minimal Activity or launcher entry to start/stop service for development.

#### Story A2 — Foreground Service implementation
- Task A2.1: Implement `ProxyForegroundService` skeleton (`onCreate`, `onStartCommand`, `onDestroy`).
- Task A2.2: Implement Notification Channel creation.
- Task A2.3: Implement persistent notification content (shows host/ports and state).
- Task A2.4: Add notification action(s) (Stop; optional Start/Restart).

#### Story A3 — Service lifecycle and resilience
- Task A3.1: Set service return mode (`START_STICKY`) and document expected behavior.
- Task A3.2: Ensure idempotent start: repeated starts do not crash.
- Task A3.3: Ensure stop closes everything cleanly (service stops and native engine stops).
- Task A3.4: Add basic state tracking (running/stopped) on Kotlin side (optional).

**Acceptance Criteria (Epic A):**
- Service can be started manually; notification appears; service remains running while app is backgrounded.
- Service can be stopped; notification disappears; no crashes on repeat cycles.
